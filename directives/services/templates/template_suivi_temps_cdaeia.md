# Template: Feuille de Suivi du Temps CDAEIA

## Instructions

Ce template permet de documenter l'allocation du temps des employés pour démontrer la conformité au critère des 75% d'activités IA qualifiantes exigé par le CDAEIA.

**Important:**
- Chaque employé éligible doit consacrer **minimum 75%** de son temps à des activités qualifiantes
- Les données doivent être conservées **6 ans** minimum
- La précision est critique - Investissement Québec peut auditer

---

## Catégories d'Activités

### ✓ Activités QUALIFIANTES (comptent vers le 75%)

| Code | Catégorie | Description | Exemples |
|------|-----------|-------------|----------|
| **IA-DEV** | Développement IA | Création de modèles, algorithmes, systèmes IA | Entraînement ML, fine-tuning, développement pipelines |
| **IA-INT** | Intégration IA | Intégration de composantes IA dans les produits | API ML, déploiement modèles, MLOps |
| **IA-DATA** | Données pour IA | Préparation et gestion des données pour l'IA | ETL, feature engineering, data pipelines |
| **IA-ANA** | Analytique avancée | Analyse prédictive et intelligence d'affaires IA | Modèles prédictifs, forecasting, segmentation ML |
| **IA-AUTO** | Automatisation intelligente | Automatisation avec prise de décision IA | RPA intelligent, workflows adaptatifs |
| **IA-SEC** | Cybersécurité IA | Sécurité basée sur l'IA | Détection anomalies, threat intelligence ML |
| **IA-NLP** | Traitement du langage | NLP et IA conversationnelle | Chatbots, analyse de sentiment, extraction |
| **IA-VIS** | Vision par ordinateur | Computer vision et traitement d'images | OCR, reconnaissance d'objets, analyse vidéo |
| **DEV-NEW** | Développement nouveau (avec IA) | Nouvelles fonctionnalités intégrant l'IA | Développement produit avec composante IA |

### ✗ Activités NON-QUALIFIANTES (ne comptent PAS)

| Code | Catégorie | Description | Exemples |
|------|-----------|-------------|----------|
| **MAINT** | Maintenance | Maintenance de systèmes existants | Corrections bugs, patches, mises à jour |
| **SUPPORT** | Support technique | Support aux utilisateurs | Helpdesk, troubleshooting, incidents |
| **ADMIN** | Administration | Tâches administratives | Réunions, emails, documentation non-technique |
| **DEV-LEG** | Développement legacy | Développement sans composante IA | Fonctionnalités traditionnelles |
| **FORM** | Formation reçue | Formation suivie par l'employé | Cours, certifications (sauf si IA) |
| **AUTRE** | Autre | Autres activités | Vacances, congés, etc. |

## Bonnes Pratiques

### Do's ✓

1. **Remplir quotidiennement** - Plus précis que de mémoire en fin de semaine
2. **Être spécifique** - "Entraînement modèle NLP pour chatbot client" plutôt que "IA"
3. **Conserver les preuves** - Commits Git, tickets Jira, documentation technique
4. **Faire valider régulièrement** - Approbation superviseur chaque semaine
5. **Archiver systématiquement** - Backup des données 6+ ans

### Don'ts ✗

1. **Ne pas arrondir excessivement** - 7.5h ≠ 8h
2. **Ne pas catégoriser vaguement** - Chaque entrée doit être claire
3. **Ne pas mélanger les catégories** - Une tâche = un code
4. **Ne pas oublier le non-qualifiant** - Réunions, admin comptent aussi
5. **Ne pas falsifier** - Pénalités IQ de 2 000$ à 25 000$
